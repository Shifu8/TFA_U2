Resumen Tecnico de la Integracion con JFlex y CUP

Proyecto: Procesamiento de Lenguaje Natural Basico | Unidad 2

1. Objetivo del Cambio

Se actualizo el backend del proyecto para reemplazar el analizador lexico y el parser escritos manualmente por analizadores generados formalmente con JFlex y CUP. Con este cambio, el programa mantiene el mismo funcionamiento visual y la misma API REST, pero el proceso interno de analisis queda basado en herramientas propias de compiladores y teoria de lenguajes.

Flujo actual: 
Oracion del usuario
  -> NormalizadorOracion
  -> LexerCup generado por JFlex
  -> ParserCup generado por CUP
  -> ResultadoAnalisis
  -> DetectorAmbiguedad
  -> API REST
  -> Frontend React

2. Cambios Realizados

  1. Se configuro Maven para generar automaticamente las clases necesarias de JFlex y CUP durante la fase generate-sources.
  2. Se creo el archivo LexerCup.flex, donde se define el analizador lexico con JFlex.
  3. Se creo el archivo ParserCup.cup, donde se define la gramatica libre de contexto con CUP.
  4. Se agrego AnalizadorJFlexCup.java para conectar el lexer generado con el parser generado.
  5. Se agrego ConstructorResultadoCup.java para construir el resultado final que consume el frontend.
  6. Se actualizo AnalizarOracionUseCase.java para usar el nuevo analizador con JFlex y CUP.
  7. Se mantuvo el detector de ambiguedad, porque esa funcionalidad trabaja despues de validar la oracion.
  8. Se eliminaron las clases manuales Lexer.java, Parser.java y Fragmento.java, ya que fueron reemplazadas por la nueva integracion.
  9. Se actualizo la documentacion del proyecto en el README.md.
  10. Se ajusto el comando del frontend para que Vite muestre un solo enlace local.

4. Funcionamiento del Analizador Lexico

El archivo LexerCup.flex define el analizador lexico. Este recibe la oracion limpia y separa cada fragmento de texto. Luego consulta la gramatica del proyecto para clasificar cada lexema.

Los tokens reconocidos son:

ART   -> articulos
PRON  -> pronombres
SUST  -> sustantivos
V     -> verbos
PREP  -> preposiciones
ADV   -> adverbios
ERROR -> lexemas no permitidos o no reconocidos

Ejemplo:

El perro corre rapido

El      -> ART
perro   -> SUST
corre   -> V
rapido  -> ADV

5. Funcionamiento del Parser Sintactico

El archivo ParserCup.cup contiene la gramatica libre de contexto que valida la estructura de las oraciones. CUP toma los tokens producidos por JFlex y verifica si pueden formar una oracion valida. 

S  -> SN SV
SN -> ART SUST
SN -> PRON
SN -> SUST
SV -> V
SV -> V C
C  -> ADV
C  -> SN
C  -> PREP SN
C  -> ADV C
C  -> SN C
C  -> PREP SN C

6. Resultado que se Devuelve al Frontend

La interfaz React no se cambio en su estructura principal, porque el backend sigue devolviendo el mismo tipo de respuesta JSON. El resultado incluye:

  • Si la oracion es valida o invalida.
  • Mensaje de estado o error.
  • Sujeto identificado.
  • Verbo identificado.
  • Complemento, si existe.
  • Tabla de tokens y lexemas.
  • Derivacion por la izquierda.
  • Arbol sintactico.
  • Informacion de ambiguedad cuando aplica.

7. Manejo de Errores

El sistema conserva mensajes claros para el usuario. Los errores lexicos se detectan cuando aparece un numero, un simbolo no permitido o una palabra que no pertenece al vocabulario. Los errores sintacticos se detectan cuando la secuencia de tokens no cumple la gramatica definida en CUP.

Ejemplos de codigos de error:

ORACION_VACIA
NUMERO_NO_PERMITIDO
SIGNO_NO_PERMITIDO
PALABRA_NO_RECONOCIDA
FALTA_SUJETO
FALTA_VERBO
ORDEN_INCORRECTO
CONCORDANCIA_INVALIDA
COMPLEMENTO_INVALIDO

10. Explicacion Corta para Sustentar

Se reemplazo el analizador manual por una integracion formal con JFlex y CUP. JFlex se encarga del analisis lexico, convirtiendo cada palabra en un token. CUP se encarga del analisis sintactico, validando esos tokens con la gramatica libre de contexto del proyecto. Luego se construye el mismo resultado que ya usaba el frontend: tabla de tokens, derivacion, arbol, sujeto, verbo, complemento y errores. De esta manera, el programa conserva la misma interfaz, pero internamente usa analizadores generados.
